# Workflow: Build Feature (WebApp Architect)

1.  **Analyze Goal**: Understand the feature requirements and how it fits into the current stack.
2.  **Scan Repo**: Use `repo_scan` to identify relevant files, API endpoints, and database models.
3.  **Design Architecture**: Use `architecture_plan` to draft the implementation strategy.
4.  **Create Plan**: Produce an `implementation-plan.md` artifact for user review.
5.  **Scaffold**: Use `scaffold_feature` to create the initial file structure and stubs.
6.  **Implement**: Iteratively fill in logic, ensuring type safety and consistency with existing patterns.
7.  **Test**: Generate and run unit/integration tests using `generate_tests`.
8.  **Verify**: Perform a manual walk-through of the flow.
9.  **Document**: Update README or docs to reflect the new feature.
